This is an example case study, not a real one. It's here so I can see how a case study is structured before writing the first real one.

## How to add a real case study

1. Copy this file into `src/content/work/` and rename it, e.g. `internal-reporting-automation.md`.
2. Fill in the front-matter. `title` is a descriptive title (no company names). `date` is roughly when the work happened (`YYYY-MM-DD`). `summary` is a one-sentence problem statement for the list card. `skills` is a short list of skills/tools involved, e.g. `["Python", "SQL", "Automation"]`. `draft` should be `false` (or removed) once it's ready to publish.
3. Write the body using the case study format: problem, what I built, result. Keep it anonymized, no company names, no code, no internal details.

This whole website counts as case study/project #1 on the Work page, separate from this collection.
