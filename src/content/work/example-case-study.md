This is an example case study, not a real one — it exists so you (Marc) can see how a case study is structured before writing your first real one.

## How to add a real case study

1. Copy this file into `src/content/work/` and rename it, e.g. `internal-reporting-automation.md`.
2. Fill in the front-matter:
   - `title` — a descriptive title (no company names).
   - `date` — roughly when the work happened (`YYYY-MM-DD`).
   - `summary` — one-sentence problem statement for the list card.
   - `skills` — a short list of skills/tools involved, e.g. `["Python", "SQL", "Automation"]`.
   - `draft` — set to `false` (or delete the line) once it's ready to publish.
3. Write the body using the case study format: **Problem** → **What I built** → **Result**. Keep it anonymized — no company names, no code, no internal details.

Remember: this whole website counts as case study / project #1 on the Work page, separate from this collection.
